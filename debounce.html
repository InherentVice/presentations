Das Problem:

_.debounce

Funktion wird ausgefüht sobald ein Event für X Millisekunden nicht mehr getriggert wurde.

Funfact: Alternatives Verhalten

            
              _.debounce(
                myFunc,
                300,
                { 'leading': true, 'trailing': false}
              )

Defaultwerte von _.debounce:


              Arguments
              func (Function): The function to debounce.
              [wait=0] (number): The number of milliseconds to delay.
              [options={}] (Object): The options object.
              [options.leading=false] (boolean): Specify invoking on the leading edge of the timeout.
              [options.maxWait] (number): The maximum time func is allowed to be delayed before it's invoked.
              [options.trailing=true] (boolean): Specify invoking on the trailing edge of the timeout.

_.throttle

stellt sicher, dass ein Event nicht öfter als alle X Millisekunden ausgeführt wird. Führt die Events aber aus, solange sie auftreten.

Direkter Vergleich _.debounce vs. _.throttle:

Perfekter Anwendungsfall für '_.throttle'

Perfekter Anwendungsfall für '_.debounce'

Fazit:

  • 'debounce' ist dafür da um eine Folge von Events abzufangen. Blockert aber alle Events, bis die Eventfolge vorbei ist.
  • 'throttle' ist geeignet, um häufig auftretende Events zu drosseln. Es kann aber den Zustand des letzten Events überspringen.
